_As of 2026-06-23 (latest AMFI NAV). Generated by `scripts/build_performance.py`._

## Positioning (data truth)
MF Pulse provides **latest-available daily NAV intelligence** + **historical performance
intelligence** + **live user-activity intelligence**. NAV is daily, not tick-by-tick — the
product never claims real-time fund data. Monthly flow data remains clearly-labelled sample.

## 4 · Returns methodology (traceable)
`return_W = (latest_NAV − NAV_W_days_ago) / NAV_W_days_ago × 100`, using the latest trading
day in a short window around each anchor. Direct vs Regular are ranked in **separate**
cohorts. `trend_score = clamp(50 + (r1m − r3m/3)·4, 0, 100)`. `quality_status` ∈ {ok, limited, stale}.

## 5 · Backfill strategy (designed; bundle-materialized today)
Returns are materialized nightly into `funds.json`. The warehouse equivalent
(`sql/003_scheme_intelligence.sql`: `fact_scheme_returns`, `fact_scheme_data_quality`,
`fact_scheme_signals` + `v_scheme_latest`) is ready for the pipeline to persist with history.
Full multi-year daily backfill (3–5y) via 90-day AMFI windows + MFAPI fallback is the next
step, gated on activating the daily service-role cron.

## 6 · Remaining source limitations
- No machine-readable AUM, expense ratio, holdings, or benchmark (factsheet/PDF only).
- 1Y returns hidden for the 5,990 schemes without sufficient history (Phase-10 rule).
- Stale/inactive schemes (5,478) excluded from the active bundle but counted here.
